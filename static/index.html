1. LangGraph Multi-Agent Orchestration

Directed Acyclic State Graph Execution

PRism models code review as a stateful graph where tasks fan out in parallel to specialized agents (Security, Quality, Logic) and converge at a Review Orchestrator node. State transitions and state merging are guaranteed transactional.

Fan-Out: Concurrent Async Workers via LangGraph
Fan-In: Merged Issue Conflict Resolution & Score Deductions

2. Tree-Sitter AST Semantic Parsing

Language-Aware Structural Understanding

Unlike simple regex matching tools, PRism builds concrete syntax trees (CST/AST) using native `tree-sitter` bindings for Python and TypeScript. This enables scope-aware analysis of function signatures, variable assignments, and SQL parameterization.

Supported Grammars: Python, TypeScript, JavaScript
Token Filtering: Extracts changed line contexts within function scopes

3. Production PostgreSQL & Redis Stack

Asynchronous Database Engine

Built with FastAPI, SQLAlchemy 2.0 (asyncpg), and Redis task queues. Utilizes connection pooling, database lifespans, and automated Alembic migrations to handle multi-tenant PR volume under peak corporate throughput.

Driver: asyncpg with pre-ping pool validation
Cache & Locks: Redis locks to prevent duplicate PR analysis

4. Webhook Security & Prometheus Observability

Enterprise Security Standard

Incoming GitHub payloads pass strict HMAC-SHA256 signature verification (`X-Hub-Signature-256`). Real-time application metrics (review counts, latency histogram, DB query counts) are exposed natively at `/metrics` for Prometheus scraping.

Security: Constant-time signature comparison
Observability: Prometheus counters, histograms, JSON logs
